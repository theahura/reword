# Noridoc: data

Path: @/data

### Overview
- Contains `puzzles.json`, the pre-generated puzzle data used by the game at runtime
- Generated offline by `@/scripts/build-words.js`; not hand-edited

### How it fits into the larger codebase

- Consumed at runtime by the game logic in `@/src/` to select daily puzzles
- Regenerated by running `npm run build-words` (defined in `@/package.json`)
- The generation pipeline filters for common-word presence but applies no size caps, so this file contains all valid puzzles from the TWL06 dictionary that have at least one common-word answer

### Core Implementation

- **JSON structure**: top-level keys are root word lengths as strings (`"3"` through `"8"`). Each value is an array of puzzle entries:
  ```json
  {
    "root": "cat",
    "expansions": {
      "s": ["cats", "cast", "scat", "acts"],
      "o": ["coat", "taco"],
      "el": ["cleat", "eclat"]
    },
    "commonKeys": ["s", "el"],
    "commonWords": ["cats", "cast", "cleat"]
  }
  ```
- Expansion keys represent the sorted extra letters added to the root. Single-character keys mean one letter was added; multi-character keys (e.g., `"el"`) mean multiple letters were added.
- No caps or truncation are applied -- all roots and all words per expansion key are preserved. This is a deliberate project invariant (see `@/.claude/CLAUDE.md`).
- Each entry also includes two common-word annotations produced by `filterByCommonWords` and preserved through `trimPuzzleData`:
  - `commonKeys`: which expansion keys contain at least one word from the top 50K common English words. Used by `getOfferedLetters()` in `@/src/game.js` to bias letter selection toward common word answers.
  - `commonWords`: the actual common words (from across all expansion keys) for this root. Used by `getAnswersForRound()` in `@/src/game.js` to sort common words before obscure ones in answer lists shown to the player.

### Things to Know

- Every root in this file is guaranteed to have at least one expansion word that appears in the 50K common English words list. Obscure-only roots are excluded during generation.
- This file is checked into version control. It should be regenerated (and the diff reviewed) whenever the build script logic or word sources change.
