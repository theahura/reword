# Noridoc: data

Path: @/data

### Overview
- Contains `puzzles.json`, the pre-generated puzzle data used by the game at runtime
- Generated offline by `@/scripts/build-words.js`; not hand-edited

### How it fits into the larger codebase

- Consumed at runtime by the game logic in `@/src/` to select daily puzzles
- Regenerated by running `npm run build-words` (defined in `@/package.json`)
- The generation pipeline filters out profane words (via `bad-words` library) and filters for common-word presence but applies no size caps, so this file contains all valid non-profane puzzles from the TWL06 dictionary that have at least one common-word answer

### Core Implementation

- **JSON structure**: top-level keys are root word lengths as strings (`"3"` through `"8"`). Each value is an array of puzzle entries:
  ```json
  {
    "root": "cat",
    "expansions": {
      "o": ["coat", "taco"],
      "el": ["cleat", "eclat"]
    },
    "trivialAnswers": ["cats"],
    "commonKeys": ["el"],
    "commonWords": ["cleat"]
  }
  ```
- Expansion keys represent the sorted extra letters added to the root. Single-character keys mean one letter was added; multi-character keys (e.g., `"el"`) mean multiple letters were added.
- Roots per length bucket are capped to 500 by `trimPuzzleData` in `@/scripts/build-words.js`, prioritizing roots with the most single-letter expansion keys. All words within each surviving root's expansions are preserved without truncation.
- Each entry includes three annotations produced during build and preserved through `trimPuzzleData`:
  - `trivialAnswers`: words that are trivial morphological inflections of the root (plurals, past tense, comparatives, superlatives, present participles). These words are already excluded from `expansions`; the two sets are disjoint. Populated at build time by `filterTrivialInflections` in `@/scripts/build-words.js` using `wink-lemmatizer`. Consumed at runtime by `isTrivialAnswer` in `@/src/game.js` to show the `'trivial-suffix'` feedback message when a player types a trivial inflection.
  - `commonKeys`: which expansion keys contain at least one word from the top 50K common English words. Used by `getOfferedLetters()` in `@/src/game.js` to bias letter selection toward common word answers.
  - `commonWords`: the actual common words (from across all expansion keys) for this root. Used by `getAnswersForRound()` in `@/src/game.js` to sort common words before obscure ones in answer lists shown to the player.

### Things to Know

- Every root in this file is guaranteed to have at least one expansion word that appears in the 50K common English words list. Obscure-only roots are excluded during generation.
- No root word or expansion word in this file is profane (as classified by the `bad-words` npm library). Profanity filtering runs early in the build pipeline, before trivial-inflection and common-word filtering. Note that runtime input validation in `@/src/game.js` provides an additional profanity check for user-typed words.
- This file is checked into version control. It should be regenerated (and the diff reviewed) whenever the build script logic or word sources change.
